# SAML2 : l’essentiel

## Quand SAML est encore utilisé
- Applications legacy / ERP / outils enterprise
- Besoins d’intégration historiques

## Différences clés vs OIDC
- XML, assertions signées
- Sessions plus orientées navigateur
- Mapping attributs via “mappers” SAML

## Stratégie typique
- OIDC par défaut pour nouvelles apps
- SAML pour legacy, avec trajectoire migration si possible
